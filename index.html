Product Overview

GoRobak dibuat untuk menaikkan visibility UMKM keliling dan membantu customer menemukan jajanan terdekat secara cepat.

Last updated: 2026
Customer

Guest bisa langsung explore

Customer tidak wajib login untuk melihat map, mencari UMKM, membuka arah Google Maps, dan melapor rekomendasi.

Vendor

Vendor harus diverifikasi

Vendor register, isi data lengkap, upload foto, lalu menunggu approval admin sebelum bisa aktif sebagai verified vendor.

Admin

Admin sebagai gatekeeper

Admin approve vendor, laporan komunitas, klaim gerobak, thread/reply, imported data, dan report UMKM bermasalah.

User Manual

Pilih role untuk melihat panduan penggunaan. Tab ini dibuat supaya dokumentasi tetap ringkas saat dibaca.

Best experience

Login sebagai customer

  • Buka halaman login dan masuk dengan akun customer.
  • Nyalakan lokasi untuk melihat posisi sendiri di map.
  • Gunakan fitur UMKM terdekat untuk menemukan vendor sekitar.
  • Simpan vendor ke favorit, lihat di peta, atau buka arah Google Maps.
  • Ikut Threads untuk share pengalaman, like, dislike, reply, dan share.
Safety

Privasi lokasi

Lokasi customer hanya dipakai di client untuk pengalaman map. Lokasi customer tidak dibagikan ke user lain dan tracking berhenti saat website ditutup.

Vendor onboarding

Register dan verifikasi

  • Daftar sebagai vendor dan isi semua data wajib.
  • Upload foto verifikasi dan menu awal.
  • Verifikasi OTP email bila flow OTP aktif.
  • Tunggu approval admin sebelum status verified.
  • Setelah verified, vendor bisa mengaktifkan status keliling.
Live operations

Dashboard vendor

  • Tekan Siap Keliling lalu Mulai untuk share lokasi vendor.
  • Pilih mode moving atau staying supaya warna marker berbeda.
  • Gunakan Stop untuk menghentikan share lokasi.
  • Catat cashflow manual: pemasukan, pengeluaran, saldo bersih.
  • Tambah menu baru melalui fitur menu dashboard.
Moderation

Admin dashboard

  • Approve atau reject vendor official.
  • Review laporan komunitas dan rekomendasi UMKM.
  • Approve thread dan reply sebelum tampil publik.
  • Delete UMKM dari report jika terbukti fake/tutup/salah.
  • Approve klaim gerobak setelah cek deskripsi dan identitas.
Import data

Google Maps import

Admin memilih area Jabodetabek, radius, dan jumlah data. Sistem mengambil data Google Places, normalisasi kategori dengan AI/heuristic, deduplicate, lalu admin bisa Accept All agar muncul di map.

Open access

Tanpa login

Guest dapat membuka Explore, menyalakan lokasi pribadi, melihat UMKM sekitar, membuka Google Maps direction, dan melapor gerobak/rekomendasi.

Upgrade experience

Kenapa login?

Login memberi fitur favorit, profil, budget/customer tools, dan pengalaman yang lebih personal tanpa menghilangkan prinsip open-source untuk UMKM sekitar.

System Design

High-level architecture GoRobak sebagai web app berbasis Next.js, Supabase, Google APIs, Brevo, dan optional Groq AI.

Client

Next.js App Router

  • Landing page
  • Explore map
  • Admin dashboard
  • Vendor dashboard
API Layer

Route handlers

  • Auth OTP/reset
  • Vendor applications
  • Threads/reactions
  • Reports/claims
Database

Supabase

  • Auth users
  • Profiles
  • Vendors
  • Products, cashflows
Integrations

External services

  • Brevo OTP email
  • Google Places
  • Google Maps direction
  • Groq normalization

Map rendering strategy

Explore page mengambil payload vendor ringan, render marker berdasarkan viewport, disable permanent tooltip massal, dan fetch detail/menu hanya saat user memilih vendor.

Realtime strategy

Realtime hanya penting untuk vendor live/visible. Update lokasi vendor di-throttle agar map tidak re-render terlalu sering di mobile.

Feature Inventory

Daftar fitur yang bisa ditinjau oleh reviewer, termasuk fitur kecil yang mendukung pengalaman customer, vendor, admin, keamanan, dan SEO.

Customer & Guest

Discovery experience

  • Explore pedagang keliling dan UMKM melalui peta.
  • Melihat posisi pengguna saat lokasi diaktifkan.
  • Mencari UMKM dan melihat UMKM terdekat.
  • Membuka detail UMKM, kategori, deskripsi, jam, area, rating, dan status.
  • Membuka arah ke warung melalui Google Maps.
  • Mengakses platform tanpa wajib login.
  • Melaporkan atau merekomendasikan gerobak/UMKM.
  • Menggunakan tombol Pakai Lokasi Ini saat melapor.
  • Mengirim report terhadap UMKM bermasalah.
Customer Login

Personal features

  • Menyimpan UMKM favorit.
  • Menghapus UMKM dari daftar favorit.
  • Mengatur budget jajan bulanan.
  • Mengakses profil customer.
  • Mobile navigation khusus customer.
  • Melihat vendor favorit langsung di peta.
Threads

Community sharing

  • Membuat thread anonymous.
  • Memilih provinsi dan nama resto/UMKM.
  • Mencari nama warung saat membuat thread.
  • Mencari thread berdasarkan nama warung atau isi cerita.
  • Filter thread berdasarkan provinsi.
  • Sort thread terbaru, terlama, paling disukai, dan paling banyak dislike.
  • Memberikan like dan dislike.
  • Share thread.
  • Reply thread dengan approval admin.
  • Warna thread mengikuti reaksi pengguna.
Vendor

Vendor operations

  • Registrasi sebagai vendor.
  • Verifikasi email menggunakan OTP.
  • Mengisi data usaha, kategori, deskripsi, area, jam, kontak, dan foto pendukung.
  • Menambahkan menu saat registrasi.
  • Menunggu approval admin sebelum aktif.
  • Mengakses dashboard setelah disetujui.
  • Mengaktifkan status Siap Keliling.
  • Membagikan lokasi saat tombol Mulai digunakan.
  • Menghentikan share lokasi melalui Stop.
  • Memilih mode bergerak atau menetap.
  • Mengaktifkan fitur SOS.
Vendor Dashboard

Management tools

  • Mengelola menu dari dashboard vendor.
  • Menambahkan menu baru.
  • Mengelola cashflow manual.
  • Mencatat pemasukan.
  • Mencatat pengeluaran.
  • Melihat ringkasan saldo bersih.
  • Mengatur target income bulanan.
  • Mengklaim data gerobak yang sudah ada.
  • Memilih toko yang ingin diklaim.

Admin

Moderation center

  • Login melalui dashboard admin.
  • Melihat vendor yang menunggu approval.
  • Membuka detail pendaftaran vendor.
  • Meninjau foto, deskripsi, area, jam, kontak, dan menu vendor.
  • Approve atau reject vendor.
  • Membuka WhatsApp vendor untuk verifikasi manual.
  • Melihat, approve, atau reject laporan komunitas.
  • Melihat thread dan reply yang menunggu approval.
  • Membedakan thread utama dan reply di dashboard.
  • Approve atau reject thread/reply.
Admin Data

Import, report, claim

  • Import data vendor dari Google Maps/Places.
  • Memilih area Jabodetabek untuk import.
  • Mengatur radius dan jumlah hasil import.
  • Melihat hasil import dan Accept All.
  • Memastikan data import muncul di map setelah diterima.
  • Melihat report UMKM.
  • Meninjau alasan report: masih exist, fake, tutup, salah lokasi, atau lainnya.
  • Mengabaikan report UMKM.
  • Menghapus UMKM dari database melalui report.
  • Approve atau reject klaim data gerobak.
  • Menghubungkan data gerobak existing ke akun vendor yang valid.

  • Reset password melalui email recovery.

Environment dan API

Catatan konfigurasi

Beberapa konfigurasi pada file environment disediakan sebagai contoh agar struktur integrasi lebih mudah dipahami. Admin atau reviewer dapat menggunakan konfigurasi dan data milik sendiri apabila ingin melakukan pengujian lebih lanjut di environment terpisah.

Beberapa fitur menggunakan API call ke layanan eksternal dan dapat memiliki limit penggunaan, seperti layanan email, peta, pencarian lokasi, atau AI. Mohon untuk tidak menggunakan fitur yang bergantung pada API eksternal secara berlebihan. Jika terdapat error pada proses tertentu, salah satu kemungkinan penyebabnya adalah limit API, konfigurasi environment, atau pembatasan dari layanan terkait.

Ketentuan Penggunaan

Penggunaan akses

Mohon untuk tidak menyalahgunakan akses admin, mengubah data di luar kebutuhan evaluasi, atau menggunakan fitur secara berlebihan. Akses ini disediakan agar reviewer dapat memahami kemampuan sistem secara menyeluruh dalam konteks penilaian lomba.

Akses admin bersifat terbatas dan dapat dicabut setelah proses penilaian selesai.

Operations Checklist

Checklist sebelum deploy production dan saat maintenance.

SEO

Domain & preview

  • Domain production: gorobak.biz.id.
  • Submit sitemap ke Google Search Console.
  • Test social preview dengan og-image.png.
  • Pastikan robots.txt tidak block public pages.
Environment

Required env

  • Supabase URL, anon key, service role key.
  • Brevo API key dan sender email.
  • OTP secret.
  • Google Places API key.
  • Groq API key jika AI normalization aktif.
Security

Safety rules

  • Never expose service role key to client.
  • Admin pages protected by role admin.
  • Customer location stays private.
  • Vendor live location stops when user stops sharing.
  • Reset password only valid through recovery session.
